<h1 align="center">安伴 AnBan</h1>

<p align="center">
  <b>一台放在家里、会主动开口关心人的 AI 陪伴设备。</b>
  <br>
  面向独居 / 居家老人——像家人一样自然聊天,会按时间和情景主动问候,
  <br>
  还把子女的牵挂带到身边:远程留言、看状态、一键发起问候。
</p>

## 它不是又一个智能音箱

|  | 普通智能音箱 | 安伴 |
|---|:---:|:---:|
| 谁先开口 | 你问它才答 | ✓ 它**主动**关心你 |
| 记得你吗 | 当成陌生人 | ✓ 记得这个人(家庭画像 + 长期记忆) |
| 和子女的关系 | 没关系 | ✓ 子女远程留言 / 看状态 / 发起问候 |
| 像不像个伴 | 冷冰冰的工具 | ✓ 有"心境",读得懂当下的氛围 |

<br>

## 三分钟看懂它能做什么

- **主动开口** — 不等指令。长时间没动静、到了该问候的点,它会自己开口,而不是干等着。
- **子女在手机上就能陪在身边** — 打开网页给老人留言(安伴用温和的语音念给老人听)、看设备在不在线 / 最近互动、一键让它替你问候一句。
- **记得这个人** — 家庭画像 + 长期记忆:知道老人叫什么、爱吃什么、上次聊到哪;越用越懂他。
- **看一眼** — 子女想知道"人在不在",可让设备拍一张、由 AI 描述当前画面(轻量视觉,不做监控)。
- **读得懂氛围(阅读心智)** — 它内部有"心境",会产生当下在意的事;子女端能"阅读"它此刻在想什么,陪伴不再是黑盒。

## 怎么用(老人零学习成本)

1. **插电开机** — 放客厅或床头,通电即用。
2. **配一次网** — 子女帮忙连一次家里 Wi-Fi(一次性)。
3. **子女扫码绑定** — 手机网页输入一个简单访问码,就能留言、看状态、发起问候。

之后老人只管像跟人说话一样跟它聊天,其余的事它自己来。

<br>

## 它怎么运转

设备只当"嘴、耳、眼",真正的"大脑"在云端——所以设备轻、稳、可换,能力可持续变强。

```
老人 ──说话 / 听 / 看──▶ [安伴设备] ──实时语音──▶ [安伴后端] ──▶ 云端 AI(听写·大模型·语音合成·看图)
                                              │
                          子女手机网页 ◀───────┘   留言 · 状态 · 主动问候 · 阅读心智
```

<br>

## 适合谁

- 子女不在身边、希望父母"有人说说话"的家庭
- 需要有人按时提醒吃药 / 喝水 / 添衣的独居老人
- 想要"看得见的牵挂",而不是打卡式冷问候的子女

<br>

## 当前阶段

安伴目前是**可路演的产品原型**:核心体验(主动语音陪伴、子女远程连接、记忆与心智)已经跑通,正在持续打磨。它**还不是**已上架的量产品。

## 数据与隐私合规

- **数据来源**:演示数据全部来自项目团队成员及其知情授权的家庭成员自测,**不含任何未经授权的第三方真实个人敏感信息**。
- **涉及数据**:老人语音与对话记录、家庭画像与长期记忆、子女账号(手机号)、以及子女主动触发「看一眼」时拍摄的单张照片。
- **使用方式**:仅用于实现陪伴对话与本次演示;摄像头**仅在子女主动触发时拍摄单张图片**,不做持续监控或录像;数据存于本项目自有云服务器。语音 / 图像会发送至上述云 API 完成识别、理解与合成,遵守各服务条款。
- **合规措施(现状与规划)**:当前为演示原型,遵循最小化采集原则,完整对话记录为开发期功能;隐私告知与知情同意、数据加密 / 脱敏、保留期限与一键删除等机制,列为产品化阶段必做项推进(本期诚实标注「待完善」)。
