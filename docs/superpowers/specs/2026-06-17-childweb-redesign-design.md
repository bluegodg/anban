# 子女端 childweb 视觉升级设计稿（B · 暖色玻璃，去模板化）

- 日期：2026-06-17
- 状态：待用户最终审阅
- 范围：childweb 前端视觉系统升级 + 产品收尾。不改后端 API、不改设备固件、不加新功能（只把现有功能做成"产品级"）。

## 1. 目标与问题

现在的 childweb 是 stitch 生成的规整 Material 稿，底子不差，但**像"没做完的 demo"**：满屏写死的假数据、假的 iOS 状态栏、可见的"未实现"按钮、以及**到处大圆角 + 处处浮卡 + 柔光团**的"AI 默认款"观感。

本次目标：在保住"安伴"暖色温度的前提下，升级为**有辨识度、像真正产品**的界面——核心是给"暖色玻璃(B)"方向**加上设计纪律**，并完成产品收尾。

## 2. 设计方向与原则（B，但有纪律）

方向 = **暖色玻璃**：暖色背景 + 半透磨砂"玻璃"主角面 + 赤陶暖棕。为避免"AI 模板感"，强制以下纪律：

1. **圆角分级**（不再一刀切 18–26px）：见 §3 尺度。
2. **每屏只有一个"玻璃主角面"浮起**（带柔阴影）；其余卡片一律**白底 + 1px 暖色细描边、压平不飘** → 层次靠"浮 vs 平"，不是"全都飘"。
3. **去柔光团**：背景只留**一道很淡的顶部暖色过渡**，不放 radial glow blob。
4. **状态标签用小方角标**（radius 4–5），不用全圆胶囊（toggle 等真正的开关控件除外）。
5. **底栏拍平**：细线分隔 + 图标，不做悬浮磨砂大圆条。
6. **无假状态栏**：全局删除假的时间/信号/电量（真 App/PWA 用系统自带）。
7. 用**字体层次 + 小字距大写栏目标题**撑结构，而非靠更多圆角卡片。

## 3. 设计 Token

### 3.1 配色（沿用现有 tailwind.config 调色，明确主用色）
- 背景 `#FAF6F0`；顶部暖色过渡 `linear-gradient(180deg, rgba(255,196,160,.30), transparent)`。
- 主色（实心主操作/品牌）`#9a4429`；暖橙渐变仅用于登录 hero 等极少处 `#EE9670→#F4B48F`。
- 玻璃面 `rgba(255,255,255,.55)` + `backdrop-filter: blur(14px)` + 边 `rgba(255,255,255,.9)`。
- 平面卡片 `#fff` + 描边 `#ece2d9`（输入/可点描边 `#e8ddd4`）。
- 文字：主 `#2a1410`、次 `#8a6f63`、弱 `#a89488`；在线/成功 `#1a8c5a`（底 `#eafaf0`）。
- 标签底色：每天 `#f5e3da`/`#9a4429`，重要 `#fbe7c4`/`#9a6a13`。

### 3.2 字体
沿用现有：正文/标签 **Be Vietnam Pro**，大标题 **Plus Jakarta Sans**。强化层次：hero 标题 24–25px/800/紧字距；栏目标题 12px/800/字距 0.14em/主色（小字距大写感）；正文 13–15px。

### 3.3 圆角尺度（关键）
| 用途 | 圆角 |
|---|---|
| 玻璃主角面 / 大容器 | 16px |
| 普通卡片 / 按钮 / 输入 | 10px |
| 图标块 | 8px |
| 状态/分类小标签 | 4–5px |
| 开关 toggle / 在线点 | 99px（控件语义需要） |

### 3.4 深度与阴影
- 仅"玻璃主角面"用柔阴影 `0 12px 30px rgba(247,140,107,.16)`。
- 其余一律**描边代替阴影**，不浮。
- 图标：生产环境用 **Material Symbols**（现有），**不用 emoji**（样稿里的 emoji 仅占位）。
- 动效：沿用现有克制过渡；只动 transform/opacity，遵守 reduced-motion。

## 4. 组件规范
- **主按钮**：实心 `#9a4429`、白字、radius 10。**次按钮**：白底 + `#e8ddd4` 描边、radius 10。
- **小标签**：方角 radius 4–5（在线/已播报/每天/重要）。
- **平面卡片**：白底 + `#ece2d9` 描边、radius 10、无阴影。
- **玻璃主角面**：见 3.1/3.4，每屏至多一个。
- **聊天气泡**：子女=实心赤陶、radius `12 12 3 12`；老人=白底描边、radius `12 12 12 3`；状态用纯文字小字（✓已播报 / ○待播报），不做胶囊。
- **输入栏**：顶部细线分隔 + 白底描边输入(r10) + 实心赤陶发送键(r10)。
- **选择器**（时间/频率）：保留现有交互，重新套系统配色与圆角。
- **底栏**：细线顶边 + 图标，激活态主色、其余弱色，拍平。
- **栏目标题**：12px/800/字距 0.14em/主色。

## 5. 各屏
**主路径（做深，按已确认样稿）：**
- **登录**：顶部暖色 hero(品牌🌿安伴+一句话) + 玻璃登录卡(访问码输入 + 实心登录键 + "新设备激活")。
- **首页**：玻璃状态卡(hero，文案接 **AnBan Mind/status** 真实数据) → 动作对(问候妈妈实心 / 看一眼描边) → 最近留言(平面卡 + 写留言引导，空状态友好) → 今日提醒摘要(替掉假"温度/湿度") → 拍平底栏。
- **消息**：标题 + 一句说明(非卡) → 气泡时间线(真实留言/回复 + 已/待播报) → 输入栏。
- **提醒**：玻璃"创建新提醒"卡(时间/频率/内容/重要开关/保存) → "我的提醒"平面卡列表(图标块 + 分类/重要方角标 + toggle)。

**其余屏（套同一系统，不逐像素打磨）：** 历史、详情、家人、我的(设置)、家人编辑。统一配色/圆角/描边/玻璃规则即可。
